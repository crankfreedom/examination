# ExamHub 架构决策日志 (ADR)

## ADR-0001
- 日期：2026-07-10
- 决策：采用模块化单体架构，不拆分微服务
- 原因：V1 团队规模小，避免分布式复杂度
- 影响：所有模块在同一进程中，通过 Service 接口通信

## ADR-0002
- 日期：2026-07-13
- 决策：采用 Chalk 采集器作为自动化内容获取方案
- 原因：V1 需要快速积累真题内容，纯手工导入效率不足
- 影响：新增采集模块，通过子进程隔离运行

## ADR-0003
- 日期：2026-07-13
- 决策：建立 Multi-Agent 开发框架
- 原因：长期维护项目需要规范化的 Agent 协作机制
- 影响：创建 .agents/ context/ tasks/ review/ prompts/ workflow/ 目录
