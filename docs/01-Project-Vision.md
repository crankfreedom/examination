# ExamHub 项目愿景与定位

> **文档版本：** v1.2.0
> **最后更新：** 2026-07-13

---

## 一、项目代号

**ExamHub** — 考试内容平台

---

## 二、多阶段定位

| 版本 | 定位 | 核心能力 |
|------|------|---------|
| **V1** | 资料库 + PDF 销售平台 | 资料浏览、搜索、购买、下载、PDF 动态生成、水印保护、下载权限管理 |
| **V2** | 在线刷题平台 | 在线答题、自动判分、答题卡、计时模拟、错题本、统计分析 |
| **V3** | 综合在线学习平台 | AI 解析、AI 组卷、AI 错题分析、个性化学习路径、智能推荐 |

---

## 三、核心设计理念

> **Content First** — 不是围绕 PDF，而是围绕题目（Content）设计整个系统。

### 含义

- 核心数据模型围绕 **Question（题目）** 设计，而非 PDF 文件
- PDF 只是内容的一种**渲染输出格式**，不是数据源
- 同一套题可以导出为 PDF、在线刷题、打印试卷等多种形态
- 系统天然支持从 V1 到 V2、V3 的平滑演进

| 用户群体 | 需求 |
|---------|------|
| 公考考生 | 国考、省考真题和模拟题 |
| 事业单位考生 | 事业单位考试资料 |

---

## 五、解决的问题

1. **资料分散**：各类考试资料分散在网盘、公众号、淘宝等渠道，查找和下载不便
2. **体验不佳**：网盘下载速度慢，资料格式混乱，无水印保护
3. **无法更新**：买的纸质书/电子版无法获取后续更新
4. **安全风险**：资料被随意转发、倒卖，缺乏知识产权保护
5. **支付门槛**：传统淘宝/微店购买流程繁琐，支付即登录体验不佳

---

## 六、产品原则

| 原则 | 说明 |
|------|------|
| **用户至上** | 简化购买流程，支付即登录，提供优质的阅读和下载体验 |
| **内容为王** | 持续丰富题库资源，保证题目准确性 |
| **开放演进** | 从 PDF 销售起步，逐步演变为综合学习平台 |
| **安全可控** | 水印保护 + 权限控制，保护内容和创作者权益 |
| **SEO 驱动** | 每个试卷、合集、商品都有独立、永久的 URL |

---

## 七、竞争壁垒

| 壁垒 | 说明 |
|------|------|
| 内容质量 | 专业录入流程，统一的 JSON Schema，标准化的答题解析 |
| 内容广度 | 覆盖多个主要考试类别，持续扩展 |
| 用户体验 | 支付即登录的流畅体验，在线阅读 + PDF 下载双模式 |
| 更新机制 | 已购用户永久免费更新，形成持续信任 |
| 价格优势 | 单卷 9.9 元起，相比纸质书和培训班有明显价格优势 |

---

## 八、商业模式

| 模式 | 收入来源 |
|------|---------|
| 单卷购买 | 每套试卷独立定价销售（如 9.9 元） |
| 套餐购买 | 多套试卷打包销售（如 100 套 99 元） |
| VIP 会员 | 包年无限下载（如 199 元/年） |
| 积分兑换 | 签到获取积分兑换资料，提升用户粘性 |

---

## 九、成功指标（V1）

| 指标 | 目标 |
|------|------|
| 试卷数量 | 上线时 ≥ 100 套 |
| 覆盖考种 | ≥ 6 种（国考/省考/事业单位/教师/考研/四六级） |
| 注册转化率 | 访客 → 注册 ≥ 15% |
| 支付转化率 | 访客 → 支付 ≥ 3% |
| 支付即登录率 | ≥ 95%（微信扫码自动注册） |
| PDF 生成成功率 | ≥ 99.5% |
