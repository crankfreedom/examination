# ExamHub 错误码规范

> **文档版本：** v1.2.0
> **最后更新：** 2026-07-13

---

## 一、错误码规范

### 1.1 格式

`
{模块码(2位)}{错误类型码(2位)}{序号(2位)}
`

| 位数 | 含义 | 范围 |
|:----:|------|------|
| 第 1-2 位 | 模块码 | 00-99 |
| 第 3-4 位 | 错误类型码 | 00-99 |
| 第 5-6 位 | 序号 | 00-99 |

**示例**：40101

| 位 | 值 | 含义 |
|:--:|:--:|------|
| 40 | 40 | 认证模块 |
| 01 | 01 | 未登录 |
| 01 | — | 序号 |

### 1.2 模块码分配

| 模块码 | 模块 |
|:-----:|------|
| 00 | 通用 |
| 10 | 内容（Question / Paper / Collection / Material） |
| 20 | 商品（Product） |
| 30 | 订单（Order） |
| 40 | 认证（Auth） |
| 41 | 用户（User） |
| 50 | 下载（Download） |
| 60 | PDF |
| 70 | 搜索 |
| 80 | 导入（Importer） |
| 90 | 管理端 |

### 1.3 错误类型码分配

| 类型码 | 含义 |
|:-----:|------|
| 00 | 通用错误 |
| 01 | 未登录 / Token 问题 |
| 02 | 权限不足 |
| 03 | 参数校验 |
| 04 | 资源不存在 |
| 05 | 资源冲突 |
| 06 | 状态错误 |
| 07 | 频率限制 |
| 10 | 业务规则 |

> **注**：所有 API 响应统一使用 HTTP 200 状态码，业务状态通过响应体中的 code 字段区分。本表中「参考 HTTP」列仅作为错误语义的分类参考，不影响实际响应状态码。
