# ExamHub PDF 中心（PDF Center）

> **文档版本：** v1.2.0
> **最后更新：** 2026-07-13

---

## 一、概述

PDF 中心负责将 Content（Paper / Collection）渲染为 PDF 文件。PDF 只是渲染结果，不是数据源。生成流程在客户端进行（前端解密 + 浏览器端生成 PDF），服务器端只做加密传输和 OSS 缓存管理。

---

## 二、核心原则

> **PDF 只是渲染结果，不是数据源。**

1. 永远由 Question → Paper → Render → 动态生成
2. PDF 不提前生成，按需生成 + 缓存到 OSS
3. 题目更新后自动清除缓存，重新生成
4. 已购用户可永久免费更新

---

## 三、PDF 生成架构

### 3.1 生成流程

\\\
┌─────────────────────────────────────────────────────────────────┐
│                        服务器端                                   │
│                                                                  │
│  数据库读取 Paper JSON                                            │
│      ↓                                                           │
│  组装完整内容（题目 + 答案 + 解析 + 水印信息）                        │
│      ↓                                                           │
│  AES-GCM 加密（密钥每次生成，随 Token 返回）                         │
│      ↓                                                           │
│  HTTPS 传输给前端                                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                        前端（浏览器端）                             │
│                                                                  │
│  接收加密数据                                                      │
│      ↓                                                           │
│  浏览器内存解密（Web Crypto API）                                    │
│      ↓                                                           │
│  HTML Render（将 JSON 格式的题目渲染为 HTML 布局）                    │
│      ↓                                                           │
│  添加动态水印（明水印 + 暗水印）                                     │
│      ↓                                                           │
│  pdf-lib / pdfmake 生成 PDF                                       │
│      ↓                                                           │
│  下载到本地                                                        │
└─────────────────────────────────────────────────────────────────┘
\\\

### 3.2 为什么选择客户端生成

| 方案 | 优势 | 劣势 | 选中 |
|------|------|------|:----:|
| 服务端生成 | 统一控制，兼容性好 | 服务器 CPU 负载高，文件传输慢 | ✗ |
| 客户端生成 | 零服务器负载，可离线，隐私保护 | 依赖浏览器能力 | ✓ |

### 4.2 排版参数

| 参数 | 值 |
|------|-----|
| 纸张 | A4 (210mm × 297mm) |
| 边距 | 上下 20mm，左右 15mm |
| 正文字号 | 10.5pt |
| 标题字号 | 14pt |
| 行距 | 1.5 倍 |
| 字体 | 思源宋体 / Noto Serif CJK |
| 代码/数字 | monospace |

### 4.3 题型渲染规范

| 题型 | 渲染方式 |
|------|---------|
| 单选题 | 题干 + A/B/C/D 选项（竖排或横排） |
| 多选题 | 题干 + A/B/C/D/E/F 选项 |
| 判断题 | 题干 + （ ） |
| 填空题 | 题干 + 下划线占位 |
| 主观题 | 题干 + 留白区域（可配置行数） |

## 五、水印系统

### 5.1 明水印

| 属性 | 值 |
|------|-----|
| 内容 | 用户名 + 手机号（部分隐藏）+ 订单号 |
| 位置 | 页面底部居中 + 背景均匀分布 |
| 透明度 | 15% |
| 旋转 | -30 度 |
| 字体大小 | 8pt（底部）/ 48pt（背景） |
| 颜色 | 灰色 |

### 5.2 暗水印

**实现方案**：

| 方法 | 说明 |
|------|------|
| 字符间距微调 | 在不可感知的范围内调整字符间距，编码用户信息 |
| 行高微调 | 在段落行高中嵌入信息 |
| 像素操作 | 在 PDF 页面特定位置插入不可见像素模式 |

**编码信息**：用户编号 + 订单号 + 下载时间（用于追溯泄露源头）

## 六、PDF 缓存策略

### 6.1 缓存流程

\\\
用户请求下载 PDF
  → 检查 OSS 是否有缓存：oss://examhub/pdf/{resourceId}/v{version}.pdf
  → 有缓存 → 直接返回缓存
  → 无缓存 → 触发 PDF 生成流程 → 上传 OSS → 返回
\\\

### 6.2 缓存键规则

`
oss://examhub/pdf/{resourceId}/v{version}.pdf
`

示例：\oss://examhub/pdf/P000001/v2.pdf\

### 6.3 缓存失效

| 触发条件 | 行为 |
|---------|------|
| 题目内容更新 | 版本号 +1，旧缓存自动失效 |
| 试卷结构变更 | 版本号 +1，旧缓存自动失效 |
| 管理员手动清除 | 删除 OSS 文件，下次请求重新生成 |
| 水印规则更新 | 不清除缓存（水印在客户端渲染时添加） |

## 七、PDF 版本管理

| 版本 | 说明 |
|------|------|
| v1 | 初始版本 |
| v2 | 内容更新后版本 |
| v3 | 再次更新后版本 |

- 每次内容更新版本号 +1
- 用户下载 PDF 时始终获取最新版本
- 下载中心显示当前版本号
- 已购用户永久免费更新

## 八、接口设计

| 接口 | 方法 | 用途 |
|------|------|------|
| /api/pdf/content/:resourceId | GET | 获取加密的试卷内容数据 |
| /api/pdf/status/:resourceId | GET | 查询 PDF 生成状态 |
| /api/admin/pdf/generate | POST | 管理端手动触发 PDF 生成 |
| /api/admin/pdf/clear-cache | POST | 管理端清除指定 PDF 缓存 |
| /api/admin/pdf/versions/:resourceId | GET | 查看 PDF 版本历史 |
