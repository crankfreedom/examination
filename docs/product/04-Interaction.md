# ExamHub 交互设计

> **文档版本：** v1.2.0
> **最后更新：** 2026-07-13

---

## 一、设计语言

ExamHub 的 UI 风格定位为**考试范 + Apple 风的简洁学习风**。

### 1.1 设计关键词

- 清晰 — 信息层级分明，减少视觉噪声
- 专注 — 以内容阅读为核心，界面退到幕后
- 专业 — 克制用色，克制动效，考试场景不花哨
- 一致 — 组件、间距、字体、色彩全系统统一

### 1.2 字体系统

| 用途 | 字体 | 字重 |
|------|------|------|
| 正文 | system-ui, -apple-system, sans-serif | Regular 400 |
| 标题 | system-ui, -apple-system, sans-serif | Semibold 600 |
| 数字/代码 | monospace, Consolas | Regular 400 |
| 字号层级 | 12 / 14 / 16 / 18 / 24 / 32 px | — |

### 1.3 间距系统

基于 4px 基数：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64 px

## 二、关键交互模式

### 2.1 支付即登录

**交互规则**：

| 步骤 | 交互 | 反馈 |
|------|------|------|
| 游客点击购买 | 弹出微信支付二维码 | 底部 sheet 从下方滑入 |
| 扫码支付 | 用户用微信扫码 | 显示"支付中…" loading 状态 |
| 支付成功 | 自动关闭 sheet | 顶部 toast："支付成功，已自动登录" |
| 自动跳转 | 2 秒后跳转下载页 | 页面过渡动画 |

**关键设计**：
- 游客支付全程不需要先注册
- 无感注册：微信返回 openid → 自动创建账号 → 生成 JWT
- 后续用户用手机号找回账号
- 降低支付漏斗流失

### 2.2 在线试看

**交互规则**：

| 状态 | 显示 | 操作 |
|------|------|------|
| 游客/注册用户 | 仅显示第 1~5 题 | 题号可点击，答案和解析可查看 |
| 第 6 题起 | 题目淡出 + 遮罩 | 点击遮罩 → 弹出购买引导 |
| 已购买/VIP | 全部题目可见 | 无限制 |

**试看遮罩设计**：
- 半透明遮罩覆盖第 6 题及后续题目
- 遮罩上显示："购买后可查看全部 {N} 道题 + 完整解析"
- 包含"立即购买"按钮
- 点击遮罩外部可滚动，但不显示题意

### 2.3 下载交互

| 状态 | 按钮文案 | 点击行为 |
|------|---------|---------|
| 未购买 | XX 元 购买 | 跳转支付页 |
| 已购买（可下载） | 下载 PDF | 触发下载 |
| 已购买（下载次数超限） | 下载次数已用完 | toast 提示 + 升级 VIP 引导 |
| VIP | 下载 PDF（VIP） | 触发下载，不计数 |
| 有新版本 | 更新（v2.0） | 重新生成并下载 |
| 下载中 | 下载中… | 显示进度条，禁用点击 |

### 2.4 搜索交互

| 阶段 | 交互行为 |
|------|---------|
| 聚焦搜索框 | 展开搜索建议（热门搜索词、搜索历史） |
| 输入中 | 去抖 300ms 后触发搜索 |
| 有结果 | 分两栏展示结果 |
| 无结果 | 显示"没有找到 '{关键词}' 的相关资料" + 推荐搜索词 |
| 筛选操作 | URL 同步更新，支持分享和浏览器回退 |

**去抖规则**：300ms 等待用户停止输入后触发，避免频繁请求。

### 2.5 在线阅读器交互

| 操作 | 交互 | 反馈 |
|------|------|------|
| 切换题目 | 点击右侧题号 / ← → 键盘 / 滑动 | 题目切换动画（淡入淡出） |
| 查看答案 | 点击"查看答案"按钮 | 答案区域展开，带平滑展开动画 |
| 收藏题目 | 点击题号旁的星标 | 星标填充，toast "已收藏" |
| 字号调整 | 顶部字号 A+ / A- 按钮 | 实时调整，大小步进 2px |
| 模式切换 | 点击专注模式 | 隐藏左右栏，仅显示题目 |
| 返回目录 | 点击"目录" | 侧边栏滑出 |

**展开/折叠**：
- 长题干默认折叠（显示"展开全文"）
- 答案区域默认收起
- 解析内容默认收起
- 点击展开带平滑动画

## 三、组件通用行为

### 3.1 加载状态

| 组件 | 加载态 | 错误态 | 空态 |
|------|--------|--------|------|
| 列表 | Skeleton 骨架屏（3~5 行） | 错误提示 + 重试按钮 | "暂无数据" + 引导 |
| 商品卡片 | Skeleton 卡片 | 灰色占位 | — |
| 图片 | 占位色 + 加载图标 | 图片错误占位 | — |
| 按钮 | Loading 旋转图标，禁用 | — | — |
| 下拉刷新 | 顶部加载指示器 | 加载失败提示 | — |

### 3.2 反馈组件

| 类型 | 样式 | 自动消失 | 位置 |
|------|------|---------|------|
| Toast | 浅色圆角，无干扰 | 2-3 秒 | 顶部居中 |
| Snackbar | 深色底 + 操作按钮 | 4 秒或手动关闭 | 底部 |
| Modal | 居中弹窗，半透明遮罩 | 手动关闭 | 居中 |
| Bottom Sheet | 从底部滑入 | 手动关闭或点击遮罩 | 底部 |
| 进度条 | 线性，百分比文字 | 完成后消失 | 局部/全局 |

### 3.3 手势操作（移动端）

| 手势 | 行为 |
|------|------|
| 左滑 | 下一题（阅读器） |
| 右滑 | 上一题（阅读器） |
| 下拉 | 刷新当前页 |
| 点击空白 | 关闭弹出层 |
| 长按 | 复制题目文本 |

## 四、暗黑模式

### 4.1 设计原则

- 所有页面需要同时适配浅色和深色主题
- 使用 CSS 变量实现主题切换
- 保证深色模式下文字对比度（WCAG AA 标准）

### 4.2 色板结构

`yaml
浅色模式:
  bg:         #FFFFFF / #F5F5F7
  text:       #1D1D1F
  textDim:    #6E6E73
  border:     #D2D2D7
  accent:     #007AFF (Apple blue)
  accentDim:  #0051D5
  success:    #34C759
  warning:    #FF9500
  error:      #FF3B30
  card:       #FFFFFF
  overlay:    rgba(0,0,0,0.4)

深色模式:
  bg:         #000000 / #1C1C1E
  text:       #F5F5F7
  textDim:    #98989D
  border:     #38383A
  accent:     #0A84FF
  accentDim:  #409CFF
  success:    #30D158
  warning:    #FF9F0A
  error:      #FF453A
  card:       #1C1C1E
  overlay:    rgba(0,0,0,0.6)
`

### 4.3 切换方式

- 跟随系统：prefers-color-scheme media query
- 手动切换：用户可手动切换主题
- 优先级：手动 > 系统
- 持久化：用户选择存储在 localStorage

## 五、响应式断点

| 断点 | 设备 | 布局 |
|------|------|------|
| < 768px | 手机 | 单栏，底部 Tab 导航 |
| 768px - 1024px | 平板 | 两栏，左侧筛选/题号 + 右侧内容 |
| > 1024px | PC | 完整布局，三栏/多栏 |

### 5.1 手机端适配规则

- 底部固定 Tab 导航（首页 / 分类 / 资讯 / 我的）
- 搜索框改为搜索图标（点击展开全屏搜索）
- 分类筛选改为底部 Sheet / 弹出层
- 阅读器改为上下滑动（非左右分页）
- 商品详情：信息堆叠，价格和购买按钮吸底

### 5.2 平板端适配规则

- 阅读器左侧显示题号目录
- 分类页左侧显示筛选面板
- 商品详情页面左右两栏布局

## 六、动画与过渡

| 场景 | 动画 | 时长 | 缓动函数 |
|------|------|------|---------|
| 页面切换 | 淡入淡出 | 200ms | ease |
| 弹窗出现 | 淡入 + 向上微移 20px | 250ms | ease-out |
| 弹窗消失 | 淡出 | 150ms | ease-in |
| 列表加载 | 逐项淡入（stagger） | 每项 50ms | ease-out |
| 按钮点击 | scale 0.97 | 100ms | ease |
| 展开/折叠 | 高度过渡 | 250ms | ease |
| 下拉刷新 | 弹簧效果 | 300ms | cubic-bezier(0.34, 1.56, 0.64, 1) |

**动画原则**：不可过度设计。动效应帮助用户理解状态变化，而非增加视觉负担。

## 七、空状态与错误状态

### 7.1 空状态

| 场景 | 文案 | 操作引导 |
|------|------|---------|
| 搜索无结果 | "没有找到 '{关键词}' 的相关资料" | "试试其他关键词" |
| 筛选无结果 | "没有找到符合条件的资料" | "建议减少筛选条件" |
| 订单为空 | "还没有订单" | "去首页挑选资料" |
| 下载为空 | "还没有下载过资料" | "去下载中心看看已购资料" |
| 收藏为空 | "还没有收藏" | "去首页发现好资料" |

### 7.2 错误状态

| 场景 | 交互 |
|------|------|
| 网络异常 | 页面顶部显示"网络似乎已断开"横幅，自动恢复后消失 |
| 请求超时 | 显示"加载超时" + 重试按钮 |
| 系统错误 | 显示"出错了，请稍后重试" + 联系客服 |
| 支付失败 | 显示错误原因 + "重新支付"按钮 |
