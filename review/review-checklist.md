# ExamHub Review Checklist

## 代码规范
- [ ] 命名是否符合规范（kebab-case / PascalCase / camelCase）
- [ ] 是否使用了 TypeScript 严格模式
- [ ] 是否存在隐式 any 类型
- [ ] 注释是否恰当（不废话，复杂逻辑有注释）

## 配置
- [ ] 配置是否直接书写在 src/config/ 模块中（不从 process.env 获取）
- [ ] 新增配置项是否通过 config/index.ts 聚合导出

## 性能
- [ ] 是否存在 N+1 查询
- [ ] 数据库查询是否使用了索引
- [ ] 是否需要缓存
- [ ] 是否有不必要的重复计算

## 安全
- [ ] 用户输入是否经过校验
- [ ] API 是否加入了认证和权限检查
- [ ] 密码是否使用 bcrypt 加密
- [ ] Token 是否设置有效期
- [ ] 是否存在 SQL 注入风险
- [ ] 是否设置了安全 HTTP 头

## SQL
- [ ] 表名/字段名是否符合命名规范
- [ ] 是否包含 created_at 和 updated_at
- [ ] 外键关系是否正确定义
- [ ] 索引是否覆盖高频查询
- [ ] Migration 是否可回滚

## API
- [ ] 响应格式是否符合统一规范（code/message/data）
- [ ] 错误码是否正确使用
- [ ] URL 路径是否使用 kebab-case
- [ ] 是否遵循 RESTful 设计
- [ ] 是否添加了参数校验

## Vue
- [ ] 组件是否使用 Composition API + script setup
- [ ] 是否使用 CSS Variables 而非硬编码
- [ ] 是否适配响应式（手机/平板/PC）
- [ ] 暗黑模式是否支持
- [ ] 是否包含加载态和空状态

## 测试覆盖率
- [ ] 关键路径是否有单元测试
- [ ] API 测试是否覆盖正常和异常流程
- [ ] 测试是否幂等可重复

## 日志
- [ ] 关键操作是否记录日志
- [ ] 错误信息是否包含足够上下文
- [ ] 日志级别是否合理

## 异常处理
- [ ] 是否捕获了可能抛出的异常
- [ ] 错误信息是否对用户友好
- [ ] 是否回滚了已变更的数据

## 文档更新
- [ ] 代码变更是否导致文档需要更新
- [ ] 是否需要创建 ADR
- [ ] API 文档是否与实现一致
